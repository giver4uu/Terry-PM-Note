## 1. MVP Use Case 실무 검증 (5개)

| 항목 | 내용 |
|------|------|
| **실무 빈도** | 주간 (주 1회 HR 리뷰 미팅 필수 아젠다) |
| **현재 해결 방법** | ATS 수동 리포트 → 엑셀 피벗 → 평균 계산 (2-3시간 소요) |

| **권장 사항** | **데이터 품질 경고 기능 필수**: "Stage 업데이트 누락 15건, 분석 신뢰도 중간" 표시 |

**보리의 코멘트:**

## 6. 추가 발굴 Use Case (보리 제안)

### UC-025: 후보자 "연락 블랙홀" 알림 ⭐⭐⭐⭐⭐

**해결하는 문제:**
후보자 Ghosting (갑자기 연락 끊김). 면접 확정했는데 당일 노쇼, 오퍼 수락 후 첫 출근 안 함. 현재는 당일에야 알아차림.

**온톨로지 활용:**
Communication 패턴 급변 감지 → "이 후보자, 평소 6시간 내 응답하다가 72시간 무응답. 유사 케이스 70%가 이후 이탈. 백업 후보 준비 권장"

**왜 제리가 놓쳤나:**
PM은 "프로세스 완결"에 집중. 실무는 "중간 이탈"이 더 고통. 특히 스타트업에서 대체 후보 없으면 채용 일정 무너짐.

**우선순위:** Phase 2 추가 권장 (구현 쉬움, 실무 가치 높음)

---

### UC-026: 면접관 "번아웃" 모니터링 ⭐⭐⭐⭐

**해결하는 문제:**
같은 면접관이 한 달에 20회+ 면접 시 피드백 품질 하락, 짜증 섞인 평가. HR이 눈치로만 파악.

**온톨로지 활용:**
Interviewer 면접 빈도 + 피드백 소요 시간 + 점수 표준편차 종합 → "김 부장 번아웃 위험, 2주 휴식 권장" 알림

**왜 제리가 놓쳤나:**
면접관 경험(Interviewer Experience)은 후순위. 실무에서는 좋은 면접관 유지가 채용 품질의 핵심. 번아웃 방치 시 "면접 거부" 선언.

**우선순위:** Phase 3 (데이터 축적 필요, 조직 문화 민감)

---

### UC-027: 공고 "퍼널 이상" 조기 경고 ⭐⭐⭐⭐⭐

**해결하는 문제:**
공고 올렸는데 지원자 10명도 안 옴. 2주 뒤에야 "공고 문제?" 파악. 시간 낭비.

**온톨로지 활용:**
유사 공고 대비 지원율 비교 → "첫 48시간 지원 3명, 평균 20명 대비 85% 낮음. 공고 타이틀/연봉 재검토 권장" 즉시 알림.

**왜 제리가 놓쳤나:**
Pre-Application은 있지만 "공고 품질" 피드백 루프는 없음. 실무에서 공고 실패는 채용 실패의 시작.

**우선순위:** Phase 2 추가 권장 (구현 쉬움, 조기 개입 = 리드타임 단축)

---

### UC-028: "레퍼런스 체크 패턴" 분석 ⭐⭐⭐

**해결하는 문제:**
레퍼런스 체크 안 하거나 형식적. 어떤 질문이 Bad Hire 예측력 있는지 모름.

**온톨로지 활용:**
"레퍼런스 체크에서 '협업 이슈' 언급 시 12개월 내 퇴사율 40% (평균 15% 대비 2.7배)" 패턴 발견. 가이드 제공.

**왜 제리가 놓쳤나:**
UC-012 (위험 시그널)에 일부 포함되지만 레퍼런스 체크 자체를 별도 Use Case로 다루지 않음.

**우선순위:** Phase 3 (레퍼런스 체크 데이터 구조화 필요)

---

### UC-029: "오퍼 협상 시뮬레이션" ⭐⭐⭐⭐

**해결하는 문제:**
"한 번 더 올려주세요" 요청. 올려줘야 할지 판단 어려움.

**온톨로지 활용:**
"이 시점에 5% 추가 제안 시 수락 확률 20%p 상승" vs "더 올려도 수락 안 할 패턴" 구분. 동적 피드백.

**왜 제리가 놓쳤나:**
UC-016 (연봉 협상 가이드)는 Phase 3인데 실무에서 더 빈번. "정적 가이드"가 아니라 "협상 진행 중 동적 시뮬레이션" 필요.

**우선순위:** Phase 2 추가 고려 (게임 이론 접근, 데이터 요구 높음)

---

## 7. 가장 큰 맹점 (보리의 종합 피드백)

### 맹점 #1: 데이터 품질 전제가 너무 낙관적

**문제:**
- 제리의 가정: "Stage Transition 90%+ 완성도"
- 보리의 현실: "실무에서 50-60%도 안 되는 곳 많음. 담당자가 업데이트 안 함"

**영향:**
온톨로지는 데이터 품질에 의존도 200%. 쓰레기 데이터 = 쓰레기 인사이트 = 고객 신뢰 상실

**대응 방안:**
- **MVP에 데이터 품질 대시보드 필수 포함**
- "이번 주 데이터 완성도: 67% (목표 80%), Stage 업데이트 누락 23건" 표시
- 누락 데이터 쉽게 보완하는 UX (클릭 몇 번으로 업데이트)

---

### 맹점 #2: 조직 정치 고려 부족

**문제:**
- 제리의 가정: "면접관에게 피드백 지연 알림 자동 발송"
- 보리의 현실: "VP/임원에게 AI가 재촉하면 HR 팀장이 혼남"

**영향:**
기술적으로 완벽해도 조직 내 권력 관계 무시하면 도입 3일 만에 기능 꺼짐.

**대응 방안:**
- **권한 레벨별 알림 로직**: 일반 직원 (자동 OK), 매니저 (HR 승인), VP+ (절대 자동 금지)
- "HR에게만 알림 → HR이 직접 전송" 워크플로우
- 톤 조정: "리마인더" (X) → "후보자 대기 중" (O)

---

### 맹점 #3: 콜드 스타트 문제

**문제:**
- 제리의 가정: "과거 데이터로 예측"
- 보리의 현실: "신규 고객은 데이터 없음. 첫 3개월이 가장 중요한데 AI 가치 못 느낌"

**영향:**
"데이터 쌓이면 좋아져요"는 고객 이탈. 첫 3개월에 가치 못 느끼면 갱신 안 함.

**대응 방안:**
- **Phase 1부터 "데이터 없을 때" 전략 필수**
- 외부 벤치마크 데이터 활용 (업계 평균)
- "당신 데이터 0건, 유사 산업 평균 제공" 명시
- 빠른 데이터 축적 유도 (게이미피케이션, 온보딩 가이드)

## 9. PM에게 한마디 (보리의 최종 조언)

### 좋았던 점

✅ **체계적인 프레임워크**: 조직별/단계별 매트릭스는 세일즈 도구로도 훌륭
✅ **온톨로지 활용 명확**: "왜 온톨로지 없이는 안 되는지" 근거 확실
✅ **AI 철학 일관 적용**: 투명성/오버라이드/학습 루프 모든 Use Case에 적용

### 부족했던 점

❌ **실무 현실과 괴리**: 데이터 품질 낙관적, 조직 정치 미고려, 콜드 스타트 간과
❌ **법적 리스크 경시**: "나중에 해결" 태도는 회사 망하는 지름길
❌ **우선순위 이상**: UC-012는 좋지만 MVP엔 위험. UC-003이 안전한 Quick Win

### 다음 단계

**이론 그만, 실전 검증 시작:**

1. **HR 3명 데려오기** (스타트업 1명, 스케일업 1명, 중견기업 1명)
2. **프로토타입 보여주기** (Figma 목업 + 샘플 데이터)
3. **핵심 질문하기**:
   - "이거 쓸래요?" (반응: "오... 신기한데?" → 실패 / "지금 당장 필요한데!" → 성공)
   - "얼마 낼래요?" (가격 민감도)
   - "어떤 기능부터?" (실제 우선순위)

**목표:**
"이론적 Use Case 카탈로그"를 "고객 검증된 로드맵"으로 전환.

**예상 결과:**
- 30% Use Case는 "생각보다 안 중요"로 판명
- 20% Use Case는 "이거 MVP에 꼭 넣어주세요" 요청
- 50% Use Case는 "Phase 2에서 보자" 유지

---

**최종 점수:**
- 제리의 카탈로그: **80점 (A-)**
- 보리 검증 반영 후 기대 점수: **95점 (A+)**

**차이를 만드는 20점:**
실무 정치 + 법적 리스크 + 데이터 현실 + 콜드 스타트 대응
