## Phase 1: 질문 패턴 초기 분석

### 발견된 주요 패턴 (5가지)

#### 패턴 1: 주간/월간 정기 보고 요청
**대표 질문:**
- "이번 주 1차 인터뷰 몇명이야, 포지션도 같이 알려줘" (페이타랩)
- "12월 4주차 공고별 진행현황 분석해줘" (게임듀오)
- "이번 주와 다음 주 채용 진행 현황을 정리하려고 해" (페이타랩)

**빈도**: 약 25% (23개/92개)
**고객사**: 페이타랩, 게임듀오 집중
**특징**:
- 주기적 반복 (매주 동일 패턴)
- 표 형식 요구 ("가로 열, 세로 열")
- 전형 단계별 구분 (사전미팅, 1차, 2차)

**현재 프레임워크 분류:**
```json
{
  "intent": "analysis",
  "cognitive": {
    "metric": ["count"],
    "dimension": ["by_stage", "by_job"],
    "temporal": ["weekly"],  // ⚠️ 현재 프레임워크에 없음!
    "explanation": ["summary"]
  }
}
```

**문제점**:
- `temporal`에 "weekly" 없음 (daily, monthly만 있음)
- "이번 주 vs 다음 주" 구분 불가

---

#### 패턴 2: 년도별/월별 비교 분석
**대표 질문:**
- "2024년도 지원자 수와 2025년 지원자수 비교해줘" (기어세컨드)
- "2024년과 2025년의 합격률 차이 보여줘" (기어세컨드)
- "작년과 올해 지원자 수를 비교해줘" (테리하우스)

**빈도**: 약 10% (9개/92개)
**고객사**: 기어세컨드, 테리하우스
**특징**:
- 경영진 보고용 (연간 실적 비교)
- 년도 단위 트렌드

**현재 프레임워크 분류:**
```json
{
  "intent": "comparison",
  "cognitive": {
    "metric": ["count", "conversion_rate"],
    "dimension": ["by_stage", "by_job"],
    "temporal": ["yearly"],  // ⚠️ 현재 프레임워크에 없음!
    "comparison": ["vs_previous_period"],
    "explanation": ["summary"]
  }
}
```

**문제점**:
- `temporal`에 "yearly" 없음

---

#### 패턴 3: 원인 분석 (트러블슈팅)
**대표 질문:**
- "자바 개발자 1차 면접 본 사람들 중에서 불합격 평가 받은 사람들 중 공통적인 내용이 뭐야?" (원티드랩)
- "불합격 86명의 주요 탈락 단계와 사유 보여줘" (게임듀오)
- "66일이나 걸린 케이스는 어떤 상황인지 보여줘" (원티드랩)

**빈도**: 약 8% (7개/92개)
**고객사**: 원티드랩, 게임듀오
**특징**:
- 문제 발견 후 원인 파악
- 주관식 평가 내용 분석 요구
- "공통점" 추출

**현재 프레임워크 분류:**
```json
{
  "intent": "diagnosis",
  "cognitive": {
    "metric": ["dropoff_rate"],
    "dimension": ["by_stage"],
    "explanation": ["root_cause"]
  }
}
```

**문제점**:
- "주관식 내용 분석" → 현재 cognitive에 없음 (텍스트 분석)
- "공통 패턴 추출" → 새로운 explanation 유형 필요

---

#### 패턴 4: 특정 지원자 조회 (Who is who)
**대표 질문:**
- "서류 합격자 누구인지 알려줘" (페이타랩, 반복)
- "그로스마케터의 해당 인원 이름이 뭐야?" (페이타랩)
- "지금 2차 인터뷰 단계에 있는 사람 누구야?" (페이타랩)

**빈도**: 약 15% (14개/92개)
**고객사**: 페이타랩 집중
**특징**:
- 단순 리스트 조회
- 익명화 문제 ("지원자 이름이 숫자라서 어떤 사람인지 알 수가 없어")

**현재 프레임워크 분류:**
```json
{
  "intent": "lookup",
  "cognitive": {
    "metric": ["count"],  // ⚠️ count가 아니라 "list"
    "dimension": ["by_stage", "by_job"]
  }
}
```

**문제점**:
- `metric`에 "list" 없음 (count는 숫자, list는 이름 목록)
- 개인정보 표시 레벨 (이름 vs 익명ID) → cognitive 필요?

---

#### 패턴 5: 유입 경로 분석
**대표 질문:**
- "유입경로별 지원자수 / 서류 합격자 수 / 서류 합격율 알려줘" (페이타랩)
- "서칭 경로로 유입된 사람 몇명이야?" (게임듀오)
- "지원 경로별 분포 알려줘" (테리하우스)

**빈도**: 약 12% (11개/92개)
**고객사**: 페이타랩, 게임듀오, 테리하우스
**특징**:
- 마케팅 효과 측정
- "원티드 vs 리멤버 vs 서칭" 비교

**현재 프레임워크 분류:**
```json
{
  "intent": "analysis",
  "cognitive": {
    "metric": ["count", "conversion_rate"],
    "dimension": ["by_source"],  // ✅ 있음!
    "comparison": ["vs_other_group"],
    "explanation": ["summary"]
  }
}
```

**문제점**:
- 분류 가능! (현재 프레임워크로 커버)

## Phase 2: 분류 애매한 질문들

### 케이스 1: 복합 Intent
**질문**: "현재 포지션 중 가장 오래 오픈되어 있는 포지션이 무엇인지 알려주고 어떤 단계에서 가장 소요시간이 길었는지 알려줘" (원티드랩)

**문제**:
- 전반부: `lookup` (어떤 포지션?)
- 후반부: `diagnosis` (왜 오래 걸렸나?)

**현재 프레임워크**: Intent는 반드시 1개

**해결 방안**:
1. Intent를 `diagnosis`로 통합 (원인 파악이 목적)
2. 또는 "compound_query" 새로운 Intent 추가?

---

### 케이스 2: 데이터 추출 요청
**질문**: "지원자 데이터를 추출하고 싶은데 어떻게 해?" (페이타랩)

**문제**:
- 이건 분석 질문이 아니라 "사용법 질문"
- 현재 Intent 7개 중 어디에도 안 맞음

**현재 프레임워크**: 분류 불가

**해결 방안**:
1. `help` Intent 추가? (시스템 사용법)
2. 온톨로지 질문 범위 밖 → 제외?

---

### 케이스 3: Raw 데이터 요청
**질문**: "raw data를 줘" (힐링페이퍼)

**문제**:
- 분석 결과가 아니라 "원본 데이터" 요구
- Intent: `lookup`? `export`?

**현재 프레임워크**: 애매함

**해결 방안**:
1. `export` Intent 추가
2. 또는 `lookup`의 하위 유형?

---

### 케이스 4: 후보자 매칭 (추천)
**질문**: "AI 효율화, 데이터 분석, 자동화, 프로젝트 기획 및 운영 등의 역량을 좀 해줄 수 있는 그로스 마케터를 찾아보려고 해. 적합한 지원자 리스트를 뽑아줄 수 있을까?" (페이타랩)

**문제**:
- 이건 분석이 아니라 "후보자 추천"
- Intent: `recommendation`? 하지만 "무엇을 해야 할지"가 아니라 "누구를 뽑아야 할지"

**현재 프레임워크**: `recommendation`이 프로세스 개선용

**해결 방안**:
1. `recommendation`의 의미 확장 (후보자 추천 포함)
2. 또는 `candidate_matching` 새 Intent 추가
3. 또는 이건 온톨로지 범위 밖 (UC-012 위험 시그널과 유사)

## Phase 3: 프레임워크 개선 필요 사항 (초기 발견)

### 1. Temporal 확장 필요
**현재**: daily, monthly, cohort, time_series, future
**추가 필요**:
- `weekly` (가장 많이 쓰임!)
- `yearly` (년도별 비교)
- `custom_range` ("12월 29일 ~ 1월 2일")

### 2. Metric 확장 필요
**현재**: count, conversion_rate, dropoff_rate, avg_duration
**추가 필요**:
- `list` (이름 목록)
- `percentage` (비율, conversion_rate와 다름)
- `trend` (증가/감소 추세)

### 3. Explanation 확장 필요
**현재**: summary, root_cause
**추가 필요**:
- `pattern` (공통 패턴 추출)
- `detail` (raw data 수준)
- `comparison_insight` (비교 후 인사이트)

### 4. 새로운 Intent 검토 필요
**현재 7개로 커버 안 되는 것**:
- `help` (시스템 사용법)
- `export` (데이터 추출)
- `candidate_matching` (후보자 추천) - 또는 recommendation 확장
