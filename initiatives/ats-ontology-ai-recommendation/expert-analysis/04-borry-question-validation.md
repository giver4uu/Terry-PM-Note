# ATS 온톨로지 질문 리스트 실무 검증

**검증자:** 보리(Borry) - 10년 경력 HR 전문가
**검증 대상:** 제리가 정리한 Use Case별 채용 담당자 질문 리스트
**검증 날짜:** 2026-01-05
**검증 기준:** 실제 채용 현장에서 나오는 질문 vs PM이 만든 질문

---

## Executive Summary

| 항목 | 평가 |
|------|------|
| **전체 정확도** | 75점 (B) - 방향은 맞지만 언어가 너무 "깔끔" |
| **진짜 질문 비율** | 60% (실무에서 정말 나오는 질문) |
| **PM 질문 비율** | 40% (논리적이지만 실제론 안 물어봄) |
| **치명적 오류** | 5개 발견 (실무에선 절대 저렇게 안 물어봄) |
| **누락된 핵심 질문** | 12개 추가 필요 |

**핵심 피드백:**
> "제리님 질문들은 **너무 정제되어 있습니다.** 실제 HR은 훨씬 더 지저분하고, 감정적이고, 구체적으로 물어봐요. 예를 들어 '프로세스 병목이 어디죠?'가 아니라 **'아니 도대체 왜 CTO는 면접 일정을 3주씩 미루는 거야?'** 이게 진짜 질문입니다."

## 1. MVP Use Cases 질문 검증

### UC-007: 프로세스 병목 진단

#### 제리의 질문 리스트
1. "왜 개발자 채용이 3개월이나 걸리죠?" ✅
2. "어느 단계에서 가장 오래 걸리는 건가요?" ✅
3. "경영진한테 뭐라고 설명해야 하죠?" ✅
4. "다른 회사들은 얼마나 걸리나요?" ⚠️
5. "CTO 일정이 문제인 것 같은데 어떻게 해야 하죠?" ✅

#### 보리의 평가

| 질문 번호 | 실무 현실성 | 평가 |
|----------|-----------|------|
| 1번 | ★★★★★ | **진짜 질문.** 경영진이 주 1회는 물어봄 |
| 2번 | ★★★☆☆ | 방향은 맞지만 표현이 **너무 정제됨** |
| 3번 | ★★★★★ | **절박한 진짜 질문.** HR의 생존 본능 |
| 4번 | ★☆☆☆☆ | **PM이 만든 질문.** 실무에선 거의 안 물어봄 |
| 5번 | ★★★★☆ | 진짜 질문이지만 **더 감정적** |

#### 실무에서 진짜 나오는 질문 (수정안)

```diff
- "어느 단계에서 가장 오래 걸리는 건가요?"
+ "서류 통과했는데 왜 2주 동안 면접 일정도 못 잡아? 어디서 막힌 거야?"

- "다른 회사들은 얼마나 걸리나요?"
+ "CEO가 '경쟁사는 2주 만에 채용한대' 그러는데 우린 왜 이래?"

- "CTO 일정이 문제인 것 같은데 어떻게 해야 하죠?"
+ "CTO가 또 면접 일정 미뤘어. 이러다가 후보자 다 떨어져 나가겠어. 어떻게 설득하지?"
```

#### 보리가 추가하는 진짜 질문 (놓친 것들)

```
6. "서류 -> 1차 면접은 빠른데 왜 1차 -> 2차는 항상 2주씩 걸려?"
7. "이번 분기 채용 목표 달성하려면 어디를 줄여야 해?"
8. "병목이 CTO 때문인 걸 알겠는데, 증거 자료가 있어야 얘기를 하죠"
9. "채용 프로세스를 2주로 줄이고 싶은데 어떤 단계를 없앨 수 있을까?"
10. "우리 팀장은 면접 후 피드백을 왜 항상 1주일씩 미뤄?"
```

## 7. 최종 권장사항

### PM에게 드리는 핵심 피드백

#### 1. 질문 언어를 "실무 투"로 바꾸세요

**Before (PM 언어):**
> "프로세스 병목이 어디죠?"

**After (실무 언어):**
> "아니 도대체 왜 CTO는 면접 일정을 3주씩 미루는 거야?"

**차이:**
- PM 언어: 정제되고, 중립적이고, 전문적
- 실무 언어: 감정적이고, 구체적이고, 절박함

---

#### 2. 질문을 3가지 레벨로 구분하세요

**Level 1: 일상 질문** (매일/매주)
- "오늘 누구한테 연락해야 하지?"
- "누구 피드백 안 받았지?"

**Level 2: 전술 질문** (월간)
- "왜 채용이 3개월이나 걸려?"
- "오퍼 거절 패턴이 뭐야?"

**Level 3: 전략 질문** (분기/연간)
- "채용 ROI를 어떻게 측정하지?"
- "우리 채용 프로세스에 편향 있어?"

**현재 문제:** 제리님은 Level 1-3을 섞어서 동일 비중으로 다룸.
**권장:** MVP는 Level 1에 집중. Level 3은 Phase 3.

---

#### 3. 페르소나별 질문 뉘앙스를 차별화하세요

**스타트업 최다인 (1인 HR):**
- "할 일이 20개인데 뭐부터?"
- "CTO 설득할 데이터 있어?"
- 특징: **생존, 속도, 혼자**

**스케일업 송지현 (HR 매니저):**
- "경영진한테 뭐라고 보고하지?"
- "면접관 기준이 왜 다 다른 거야?"
- 특징: **표준화, 팀 관리, 데이터**

**중견기업 박예린 (채용 담당):**
- "이거 법적으로 문제 안 돼?"
- "편향 있는지 체크해야 하는데..."
- 특징: **리스크 관리, 컴플라이언스**

---

#### 4. 다음 단계 액션

**즉시 (P0):**
1. **질문 리스트 전면 개편**
   - 보리 피드백 반영
   - "실무 투" 언어로 재작성
   - 페르소나별 Top 5 질문 명확화

2. **누락 질문 12개 Use Case 매핑**
   - 기존 Use Case에 추가할 것
   - 신규 Use Case 필요한 것 (3-4개)

**단기 (P1 - 1주일 내):**
3. **HR 3명 인터뷰**
   - 스타트업/스케일업/중견기업 각 1명
   - 질문 리스트 보여주고 "이거 실제로 물어봐요?" 검증
   - 실제로 쓰는 표현 녹취

4. **질문 -> UI 프롬프트 매핑**
   - "왜 CTO는 면접을 3주씩 미뤄?" -> 시스템이 어떻게 답할지
   - 각 질문당 샘플 답변 시나리오 작성

**중기 (P2 - 2주일 내):**
5. **데모용 질문 Top 10 선정**
   - Wow moment 보장
   - 3가지 페르소나별 각 3-4개
   - 세일즈 훅으로 활용

6. **질문 기반 Product Hunt 태그라인**
   - "채용 담당자가 매일 물어보는 20가지 질문에 AI가 답합니다"
   - 질문 리스트가 곧 마케팅 메시지
